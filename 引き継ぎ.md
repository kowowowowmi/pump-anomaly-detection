## 作業ログ

### EDA（01_eda.ipynb）

**主な発見：**
- `sensor_15`：全行欠損 → ドロップ対象
- `sensor_00`（10,208欠損）、`sensor_50`（77,017欠損）など欠損が多いセンサーあり
- BROKEN = 7行のみ → 教師あり分類は不可。異常検知アプローチが必要
- BROKEN と欠損タイミングに相関なし

**センサー感度ランキング（目視 + 変動比分析）：**
- 一軍：sensor_42, 41, 39, 04
- 二軍：sensor_38, 43, 07, 06
- 三軍：sensor_08
- 四軍：sensor_40, 05

### ARIMA モデリング（02_modeling.ipynb・03_modeling_advanced.ipynb）

**設計判断のログ：**

1. **STL 分解は不採用**
   ACF を確認したところ、4センサー全てでラグ1440（日周期）付近にピークなし → 周期性なし → STL不要

2. **ARIMA(1, 0, 0) を採用**
   - PACF がラグ1後にカットオフ → AR(1)
   - ADF検定で全センサーp値≈0.000 → 定常（d=0）
   - ar.L1 ≈ 0.986（1に近い強い持続性）

3. **2種類の検知シグナル設計**
   - **シグナルA**（スパイク検知）：`|residual| > k × σ_normal`
     → BROKEN 前後の急激な変化を捉える
   - **シグナルB**（固着検知）：`rolling_std(residual, window=50) < k × σ_normal`
     → RECOVERING の「残差が異常に小さい状態が続く」を捉える
     ※ シグナルBの根拠：AR(1)はφ≈1のため異常水準に速やかに適応する。絶対値ではなく残差分散で検知する方が、「NORMAL水準で安定しているRECOVERING」にも対応できる

4. **4センサーOR統合の結果**

| | Precision | Recall | F1 |
|---|---|---|---|
| sensor_04 単体 | 0.91 | 0.86 | 0.89 |
| 4センサーOR統合 | 0.85 | 0.94 | 0.89 |

### Isolation Forest + アンサンブル（03_modeling_advanced.ipynb）

**設計判断のログ：**

- 使用センサー：`['sensor_04', 'sensor_39', 'sensor_41', 'sensor_42']`
- `X_normal`（NORMALのみ）で `.fit()`、`X_all`（全期間）で `.predict()`
- `contamination=0.015`（実験で調整）、`random_state=42`
- contamination は「X_normal データのうちの何%点を閾値にするか」という値。全体の異常比率とは直接対応しない点に注意
- sklearn の `.predict()` は `-1`（異常）/ `1`（正常）を返す → `(pred == -1).astype(int)` で 0/1 に変換

### LSTM-AE（04_lstm_ae.ipynb）

**実装内容：**
- PyTorch で Encoder-Decoder 構造の LSTM-AE を実装（hidden_size=64、単層、20エポック）
- スライディングウィンドウ（window_size=60、stride=1）で整形
- NORMAL 連続区間ごとに窓を作成（区間をまたぐ窓を防ぐ）
- StandardScaler で正規化（NORMAL データのみで fit）
- 全期間の復元誤差を計算し、95パーセンタイル閾値で異常判定

**設計上の気づき：**
- `mean + k×std` 閾値は NORMAL データの遷移窓（NORMAL/RECOVERING 境界付近の窓）が外れ値として std を膨らませ、閾値が高くなりすぎて Recall が極端に低くなった
- 95パーセンタイル閾値に切り替えることで外れ値の影響を排除し大幅改善
- RECOVERING 平均誤差（4.73）が NORMAL 平均誤差（0.10）の約48倍あり、モデル自体は正しく特徴を捉えていた

## 最終比較表

| 手法 | Precision | Recall | F1 |
|---|---|---|---|
| ARIMA 4センサーOR | 0.85 | 0.94 | 0.89 |
| Isolation Forest（contamination=0.015） | 0.82 | 0.98 | 0.89 |
| ARIMA + IF アンサンブル（OR） | 0.78 | 1.00 | 0.87 |
| LSTM-AE（20エポック、95パーセンタイル閾値） | 0.53 | 0.81 | 0.65 |

**考察：**
- ARIMA と IF がほぼ同等（F1 0.89）で最も良い結果
- アンサンブル（OR）は Recall 1.00 を達成したが Precision が低下。OR 統合は構造上 Recall が上がり Precision が下がるトレードオフがある
- LSTM-AE は最小構成（20エポック）では他の手法に及ばなかった。チューニング余地は大きい
