### ARIMA モデリング（02_modeling.ipynb・03_modeling_advanced.ipynb）

**設計判断のログ：**

1. **STL 分解は不採用**  
   ACF を確認したところ、4センサー全てでラグ1440（日周期）付近にピークなし → 周期性なし → STL不要

2. **ARIMA(1, 0, 0) を採用**  
   - PACF がラグ1後にカットオフ → AR(1)
   - ADF検定で全センサーp値≈0.000 → 定常（d=0）
   - ar.L1 ≈ 0.986（1に近い強い持続性）

3. **2種類の検知シグナル設計**  
   - **シグナルA**（スパイク検知）：`|residual| > k × σ_normal`  
     → BROKEN 前後の急激な変化を捉える  
   - **シグナルB**（固着検知）：`rolling_std(residual, window=50) < k × σ_normal`  
     → RECOVERING の「残差が異常に小さい状態が続く」を捉える  
     ※ シグナルBの根拠：AR(1)はφ≈1のため異常水準に速やかに適応する。絶対値ではなく残差分散で検知する方が、「NORMAL水準で安定しているRECOVERING」にも対応できる

4. **4センサーOR統合の結果**

| | Precision | Recall | F1 |
|---|---|---|---|
| sensor_04 単体 | 0.91 | 0.86 | 0.89 |
| 4センサーOR統合 | 0.85 | 0.94 | 0.89 |

OR統合でRecallが0.86→0.94に向上。ポンプ故障文脈では「見逃し > 誤報」なのでRecall重視が正しい方向。

## 次のセッションでやること

### Step 1：Isolation Forest の実装（03_modeling_advanced.ipynb に追記）

IFは「孤立しやすい点 = 異常」というアルゴリズム。
ARIMAと違い、値の絶対水準で異常を判定するため、RECOVERING のような「正常水準で安定しているが実は異常」を検知できる可能性がある。

**実装の流れ：**
```python
from sklearn.ensemble import IsolationForest

features = ['sensor_04', 'sensor_39', 'sensor_41', 'sensor_42']
X_normal = df[df['machine_status'] == 'NORMAL'][features].interpolate()
X_all = df[features].interpolate()

clf = IsolationForest(contamination=0.065, random_state=42)
# contamination ≈ RECOVERING比率（6.6%）を目安に設定
clf.fit(X_normal)
pred = clf.predict(X_all)  # -1: 異常, 1: 正常
```

`contamination` パラメータの意味と設定根拠を理解してから実装する。

**評価：**
```python
y_pred_if = (pred == -1).astype(int)
print(classification_report(y_true, y_pred_if))
```

### Step 2：ARIMA + IF のアンサンブル（OR）

```python
anomaly_ensemble = all_anomaly_bool | (pred == -1)
print(classification_report(y_true, anomaly_ensemble.astype(int)))
```

### 何をやったか

Kaggle の「Pump Sensor Data」（工業用ポンプの52センサー、220,320行、1分間隔）を使って、多変量時系列異常検知パイプラインをゼロから構築した。

EDA では52センサーの欠損・クラス分布・タイムライン可視化を行い、BROKEN が7行しかない極端な不均衡を確認。教師あり分類が使えないことを自分で導き出し、異常検知アプローチに切り替えた。

ACF/PACF・ADF検定でセンサーの時系列構造を分析し、ARIMA(1,0,0) を採用。NORMAL期間のみで学習して全期間に適用し、残差を使った2種類のシグナル（スパイク検知 / 固着検知）を設計した。4センサーのOR統合でRecall 0.94 / F1 0.89を達成した。

その後、Isolation Forestを比較手法として実装し、ARIMA + IF アンサンブルで最終評価を行った。（← 次セッションで完成させる）

### 身についたこと（技術・コード）

- **時系列の定常性検定**（ADF検定）：単位根の概念、p値による判定
- **ACF / PACF の読み方**：AR次数の決定、季節性の有無の判断
- **ARIMA(p, d, q)**：モデルの数式（漸化式として理解）、statsmodels での実装
- **残差ベース異常検知**：スパイク検知（シグナルA）と固着検知（シグナルB）の設計
- **Isolation Forest**：孤立スコアによる異常検知、contamination パラメータ
- **アンサンブル（OR/AND）**：Precision/Recall トレードオフとの関係
- **classification_report の読み方**：不均衡データでの Accuracy の限界、Recall重視の判断

### 身についたこと（理論・概念）

- **ARIMA の限界**：φ≈1の系列では異常水準に速やかに適応し、残差が消える問題。解決策として「残差分散の異常な小ささ」を検知する発想を自力で提案
- **クラス不均衡と評価指標**：BROKEN=7行では教師あり学習が成立しない理由、Precision/Recall トレードオフの実務的な意味
- **異常検知の2つのアプローチ**：時系列パターンからの逸脱（ARIMA）vs. 正常データ群からの距離（IF）が補完関係にある

### 採用しなかった選択肢とその理由

- **STL分解**：ACFで周期性が確認されなかったため不採用
- **教師あり分類（Random Forest等）**：BROKEN=7行では学習サンプルが足りない
- **単一センサー**：sensor_04 単体F1=0.89に対し、4センサー統合でRecall 0.94に向上。多センサー統合の有効性を確認
